## 基本概念
> 大部分内容的原理内容都是直接参考来自：[https://github.com/datawhalechina/hello-agents](https://datawhalechina.github.io/hello-agents/#/)
> 所有的测试代码：[agent-learn.ipynb](https://github.com/shangxiaaabb/ProjectCode/tree/main/code/Python/Agent/agent-learn.ipynb)

### Agent运行机制
智能体[^1]并非一次性完成任务，而是通过一个持续的循环与环境进行交互，这个核心机制被称为 智能体循环 (Agent Loop)
![](https://s2.loli.net/2025/11/11/wjJg7lPMS8DnBYG.webp)
换言之一个最基本的案例就是：用户通过输入（比如：今天天气如何？），Agent通过thought分析任务（需要调用什么工具，获取地理位置、获取位置天气）然后将得到的内容再输出给用户。
### ReAct
通过将思考与行动组合。思考指导行动，而行动的结果又反过来修正思考，其工作流程为:
![](https://s2.loli.net/2025/11/11/VBp73kqOhKCdP51.webp)
在上述过程中对于关键词解释如下：**1、Thought (思考)**： 这是智能体的“内心独白”。它会分析当前情况、分解任务、制定下一步计划，或者反思上一步的结果。**2、Action (行动)**： 这是智能体决定采取的具体动作，通常是调用一个外部工具，例如 `Search['华为最新款手机']`。**3、Observation (观察)**： 这是执行Action后从外部工具返回的结果，例如搜索结果的摘要或API的返回值。
智能体将不断重复这个 Thought -> Action -> Observation 的循环，将新的观察结果追加到历史记录中，形成一个不断增长的上下文，直到它在Thought中认为已经找到了最终答案，然后输出结果。这个过程形成了一个强大的协同效应：推理使得行动更具目的性，而行动则为推理提供了事实依据。
### Plan-and-Solve
对于Plan-and-Solve直接任务处理过程分为两阶段：
**1、规划阶段 (Planning Phase)**： 首先，智能体会接收用户的完整问题。它的第一个任务不是直接去解决问题或调用工具，而是将问题分解，并制定出一个清晰、分步骤的行动计划。这个计划本身就是一次大语言模型的调用产物。
**2、执行阶段 (Solving Phase)**： 在获得完整的计划后，智能体进入执行阶段。它会严格按照计划中的步骤，逐一执行。每一步的执行都可能是一次独立的 LLM 调用，或者是对上一步结果的加工处理，直到计划中的所有步骤都完成，最终得出答案。
![](https://s2.loli.net/2025/11/11/S5emRQbWgrYahjs.webp)
## 进阶概念
### 记忆系统以及检索增强
检索增强生成（RAG）是一种结合了信息检索和文本生成的技术。它的核心思想是：在生成回答之前，先从外部知识库中检索相关信息，然后将检索到的信息作为上下文提供给大语言模型，从而生成更准确、更可靠的回答。
因此，检索增强生成可以拆分为三个词汇。检索是指从知识库中查询相关内容；增强是将检索结果融入提示词，辅助模型生成；生成则输出兼具准确性与透明度的答案。
## Agent开发
就目前市面上的框架有很多如LangGraph/LangChain等，除此之外还有很多的低代码平台（基本上就和构建流程图一样）如Coze[^7]、n8n[^8]、dify[^6]等。但是对于构建一个Agent从代码角度来说复杂度不是很高。以框架LangGraph为例进行开发一个Agent。**首先简单了解Langgraph**，把AI 应用拆成 节点（Node），用 边（Edge） 控制流转，状态（State）在整个图中传递。核心就是3个概念：**1、node**：一个可执行的函数，接收 state → 返回 state 更新；**2、state**：整个工作流的共享数据结构（字典）；**3、edge**：控制流：普通边、条件边、循环边；比如说一个最简单例子（LLM查询天气）：
```python
import os
import requests
from typing import TypedDict, Annotated, List
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage, BaseMessage, ToolMessage, AIMessage
from langchain_core.tools import tool
from langgraph.graph import StateGraph, END
from langgraph.prebuilt import ToolNode

os.environ["DOUBAO_API_KEY"] = "xxxxxxxx"

# ================== 2. 天气 Tool ==================
@tool
def get_weather(city: str) -> str:
    """查询指定城市的当前天气（返回中文描述）"""
    # 心知天气 (免费，注册 https://www.seniverse.com/ 获取 key)
    API_KEY = "xxxxxx"
    url = f"https://api.seniverse.com/v3/weather/now.json"
    params = {
        "key": API_KEY,
        "location": city,
        "language": "zh-Hans",
        "unit": "c"
    }
    try:
        resp = requests.get(url, params=params, timeout=5)
        data = resp.json()
        if "results" in data:
            r = data["results"][0]
            loc = r["location"]["name"]
            weather = r["now"]["text"]
            temp = r["now"]["temperature"]
            return f"{loc} 当前天气：{weather}，温度 {temp}℃"
        else:
            return f"抱歉，未能查询到 {city} 的天气信息。"
    except Exception as e:
        return f"天气查询出错：{str(e)}"

tools = [get_weather]
tool_node = ToolNode(tools)

# ================== 3. 状态定义 =================
class AgentState(TypedDict):
    messages: Annotated[List[BaseMessage], "对话历史"]

# ================== 4. LLM 初始化（==================
# 
llm = ChatOpenAI(
    model="doubao-seed-1-6-250615",  # ['doubao-seed-1-6-250615', 'doubao-seed-1-6-flash-250615', 'doubao-seed-1-6-250615', 'doubao-1-5-lite-32k-250115']
    api_key=os.getenv("DOUBAO_API_KEY"),  # 用豆包 key
    base_url="https://ark.cn-beijing.volces.com/api/v3",  # 豆包 base URL
    temperature=0.3,
    max_tokens=512,
)
llm_with_tools = llm.bind_tools(tools)

# ================== 5. Agent 节点 ==================
def agent(state: AgentState) -> dict:
    """核心 Agent：根据历史决定是回复还是调用工具"""
    messages = state["messages"]
    response = llm_with_tools.invoke(messages)
    return {"messages": [response]}

# ================== 6. 条件路由 ==================
def should_continue(state: AgentState):
    """判断最后一条消息是否包含 tool_calls"""
    last_msg = state["messages"][-1]
    if isinstance(last_msg, AIMessage) and getattr(last_msg, "tool_calls", None):
        return "tools"      # 去执行工具
    return END              # 直接结束

# ================== 7. 构建 Graph ==================
graph = StateGraph(AgentState)

graph.add_node("agent", agent)
graph.add_node("tools", tool_node)

graph.set_entry_point("agent")
graph.add_conditional_edges(
    "agent",
    should_continue,
    {
        "tools": "tools",
        END: END
    }
)
graph.add_edge("tools", "agent")   # 工具执行完再回 Agent

app = graph.compile()
# ================== 8. 交互入口 ==================
def ask_weather(question: str, thread_id: str = "default"):
    config = {"configurable": {"thread_id": thread_id}}
    result = app.invoke(
        {"messages": [HumanMessage(content=question)]},
        config
    )
    final_msg = [m for m in result["messages"] if isinstance(m, AIMessage)][-1]
    print(final_msg.content)

# if __name__ == "__main__":
#     ask_weather("北京今天天气怎么样？")
```

## 向量数据库
向量数据库[^2][^3]是一种将数据存储为高维向量的数据库，高维向量是特征或属性的数学表示。每个向量都有一定数量的维度，根据数据的复杂性和粒度，可以从数十到数千不等。通常是通过对原始数据(如文本、图像、音频、视频等)应用某种转换或嵌入函数来生成的。嵌入函数可以基于各种方法，如机器学习模型、词嵌入和特征提取算法。**主要优点是**：它允许**基于数据的向量距离或相似性进行快速和准确的相似性搜索和检索**。这意味着不用使用基于精确匹配或预定义标准查询数据库的传统方法，而是可以使用向量数据库根据语义或上下文含义查找最相似或最相关的数据。（也就是说将文本/图片等转化为向量形式存储到数据库中，并且通过计算相似度方式进行索引检索）
向量数据库类别也有很多[^4]：Milvus、Chroma、Pinecone等。直接使用Milvus数据库：直接在远程服务器上本地部署Milvus数据库：
```bash
wget https://github.com/milvus-io/milvus/releases/download/v2.4.0/milvus-standalone-docker-compose.yml -O docker-compose.yml

# 启动服务（后台运行）
docker compose up -d
# 检查状态（应看到 3 个容器：milvus-etcd、milvus-minio、milvus-standalone）
docker compose ps # 检查状态
docker compose logs milvus-standalone # 查看日志
docker compose down   # 停止
docker compose up -d  # 重启
```
服务器安装完毕直接直接通过本地连接方式进行连接，对于数据库一般而言就是3个操作：增、删、查、改。Milvus中4类操作: [milvus_base_use.py](../../../code/Python/Agent/milvus_base_use.py)，本地连接Milvus数据库可以直接通过工具：[Attu](https://github.com/zilliztech/attu)
